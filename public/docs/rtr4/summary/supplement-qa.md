# 补充：常见疑问澄清

> 基于 Ch8-11 阅读过程中的 Q&A 整理。

---

## 1. $L_o$ 和 BRDF $f$ 是标量还是向量？

- **光谱域（正确做法）**：两者都是**标量**——每个波长 $\lambda$ 独立计算。完整颜色是这些标量在可见光谱上的分布。
- **RGB 渲染（工程实践）**：两者都是 **RGB 三元组（向量）**，每个通道独立计算。BRDF 返回 RGB 是因为 $F_0$ 和 $\rho_{ss}$ 都是 RGB 值。

Ch9 §9.3 原文："实时渲染器一般将光谱分布表示为一个 RGB 三元组，这意味着 BRDF 会返回一个 RGB 值。"

---

## 2. 光谱渲染 vs RGB 渲染的区别

不是"BRDF 多一个 $\lambda$ 参数"的区别。真正的区别在**执行顺序**：

| | 光谱渲染（正确） | RGB 渲染（工程捷径） |
|---|---|---|
| 计算 | 先**逐波长**做 $s(\lambda) \cdot r(\lambda)$ | 先积分变 RGB，再 R×R, G×G, B×B |
| 最后 | 用 $\bar{x}\bar{y}\bar{z}$ 积分 → XYZ → RGB | 已经是结果 |
| 数学 | $\int f(\lambda)L(\lambda)\bar{x}(\lambda)d\lambda$ | $(\int f\bar{x}) \cdot (\int L\bar{x})$ |

$\int fL \neq \int f \cdot \int L$，数学上不合法。但大部分材质和光源光谱够平滑时误差不大。

极端的反例**（Ch8 图8.10）**：激光投影屏的反射率只在投影仪原色波长处有尖峰（窄带高反射），而 D65 日光的能量分布在完全不同区域。RGB 渲染完全捕捉不到这种错位——投影仪和日光照射的颜色一样，但物理上反射率差 4 倍。激光投影屏反射率不平滑，它是尖峰，所以 RGB 渲染翻车。D65 日光平滑 → RGB 计算对它是对的；但激光投影屏的反射率不平滑 → RGB 计算对它也是按照"平滑"来处理的 → RGB 的"平稳假设"对它不成立 → 翻车。

## 3. 金属度参数化和 Schlick 近似的关系

**没有直接关系**——通过 $F_0$ 咬合在一起：

```
金属度参数化 ──→ F₀ ──→ Schlick 近似 ──→ 任意角度的菲涅尔反射率
（材质编码方案）    （垂直入射反射率）   （F₀ → F(θ)）
```

- **金属度参数化**负责**产生** $F_0$：$m$ 决定 $F_0$ 取 baseColor（金属）还是 0.04（非金属）。资产层面的编码方案。
- **Schlick 近似**负责**消费** $F_0$：给定 $\mathbf{n}\cdot\mathbf{l}$ 和 $F_0$，算出实际反射率。物理层面的公式。

可以独立使用——不搞金属度照样用 Schlick；金属度也能搭配精确菲涅尔方程。

## 4. BRDF 的结构

$$f = f_{spec} + f_{diff}$$

| 项 | 控制因子 | 本质 |
|----|---------|------|
| $f_{spec}$（镜面） | $F$（菲涅尔，Schlick）、$G_2$（遮蔽-遮挡，Smith）、$D$（NDF，GGX/Beckmann/Blinn-Phong） | 表面反射 |
| $f_{diff}$（漫反射） | $\rho_{ss}$（次表面反照率） | 局部次表面散射 |

- **Blinn-Phong 是 NDF**——它和 Beckmann、GGX 都是 $D(\mathbf{m})$ 的具体形式，只是计算最简单但没有形状不变性。
- **$R_{spec}$ 不是 diffuse 的本质参数**——只在耦合 diffuse（Kelemen-Szirmay-Kalos）中出现，用于能量守恒修正。Lambertian diffuse 完全不涉及 $R_{spec}$。
- diffuse 和 specular **可以独立选择组合**（Lambertian + GGX 是最常见的组合）。
- diffuse 的简单形式（Lambertian）与 $\mathbf{l}$、$\mathbf{v}$、$\mathbf{n}$ 都无关。更精确的耦合/粗糙模型则有关——也符合 Helmholtz 互易性。

## 5. 逆反射（Retroreflection）

**定义**：光更倾向于被反射回**它来的方向**。

```
普通镜面反射： 光 →   ← 反射（远离入射方向）
逆反射：       光 →   → 回到光源附近
```

**成因**（Ch9 图9.29）：当光照方向和观察方向不同时，微观几何的 shadowing 让你只能看到被遮蔽的暗区；当两者相近时，亮区和可见区重叠。

**需要关心吗？**
- 大多数游戏材质没必要——Lambertian diffuse 就够，肉眼不可见
- 例外：月球/行星表面（从地球看石头也算"微表面"）、极端粗糙织物（天鹅绒、毛毡）

皮肤需要的是**全局次表面散射**（散射距离 > 像素），不是逆反射——两个问题不要混。

---

## 6. 散射距离的两个判断维度

两个问题的判断标准是不同的串联步骤：

```
散射距离 > 像素？
  ├── 是 → 全局次表面散射（Ch14）。结束。
  └── 否 → 散射距离 < 微表面不规则性？
              ├── 是 → 粗糙 diffuse（有逆反射），如 Oren-Nayar
              └── 否 → 光滑 diffuse（无逆反射），如 Lambertian
```

第一步决定"算多细"，第二步决定"用哪个局部模型"。

**散射距离** = 光进入物体内部后在散射/吸收前的平均自由程。

具体例子：
- **皮肤**：散射距离毫米级。特写像素小于散射距离 → 全局方案
- **塑料**：散射距离极小。局部模型够，逆反射也弱 → Lambertian 就行
- **月球**：地球上观测，散射距离 < 石头尺寸 → 有逆反射

## 7. Metalness 可以取 0-1 中间值吗？

**可以。** 物理上不存在"半金属"，但工程中有意义：

```
m=0（非金属）：F₀=0.04, ρ_ss=baseColor
0<m<1：       F₀=lerp(0.04, baseColor, m), ρ_ss=lerp(baseColor, black, m)
m=1（金属）：  F₀=baseColor, ρ_ss=(0,0,0)
```

中间值用于蒙版边缘抗锯齿、部分氧化的金属（锈斑与纯金属过渡区）、非真实感材质。

0.2–0.45 范围的 $F_0$ 对应半导体（晶体硅），现实中极罕见。大面积中间值不物理。

## 8. NDF、Roughness、Normal Map 三者的关系

```
Normal Map ──→ NDF 的中心（微表面平均往哪个方向偏）
Roughness  ──→ NDF 的宽度（微表面有多分散）
NDF        ──→ 一个数学分布函数 D(m)，以 normal 为中心、以 roughness 为宽度
```

- **NDF ≠ roughness**。NDF 是函数，roughness 是参数。
- **GGX 是函数，不是查找表**。公式只有一行，比 Beckmann 还简单（无需 $\exp$），直接在着色器里算：

```hlsl
float GGX_D(float NoH, float a) {
    float a2 = a * a;
    float denom = (NoH * NoH) * (a2 - 1.0) + 1.0;
    return a2 / (PI * denom * denom);
}
```

- 不同粗糙度值对应不同的 NDF 形状——不需要换函数，只需要改 $\alpha_g$。

## 9. GGX 需要哪些纹理参数？

| 参数 | 来源 | 作用 |
|------|------|------|
| $\alpha_g$ | roughness 纹理（灰度），通常 $\alpha_g = r^2$ | → $D$（NDF 宽度） |
| $F_0$ | metalness + baseColor 推导 | → $F$（菲涅尔） |
| $\mathbf{n}$ | normal map | → $G_2$ 积分方向、$D$ 中心 |

**最小 PBR 材质定义**：

| 纹理/参数 | 作用 |
|-----------|------|
| baseColor（RGB） | 兼作 $F_0$ 或 $\rho_{ss}$（由 metalness 解释） |
| metalness（标量） | 决定 baseColor 解读为 $F_0$ 还是 $\rho_{ss}$ |
| roughness（标量） | $\alpha_g$ |
| normal（RGB） | $\mathbf{n}$ |

这个最小集合在"光很简单"（精确光源 + $L_i$ 已知）的前提下，就能完整定义材质。
**但 Ch10-11 正是在解决"光很简单"这个前提本身不够用的问题**——面光源、环境光、间接光、遮挡等。

## 10. "过滤时的联动"是什么

**问题**：物体远离时，屏幕一个像素覆盖多个法线纹素。mipmap 对法线做平均 → 分散度信息丢失 → 粗糙度没变 → NDF 过窄 → 高光闪烁锯齿。

**解法**：法线 mipmap 时额外计算方差 → 运行时用方差修正 roughness：

```
原本四个法线各指不同方向 → 有效 NDF 很宽
mipmap 平均后只剩一个方向 → 必须增大 roughness 补偿丢失的分散度
```

**工业实践**（RTR4 §9.13.1 确认）：

| 游戏/引擎 | 方法 |
|-----------|------|
| **《使命召唤：黑色行动》** | 方差映射——预计算法线方差 → 修正 roughness mipmap |
| **《教团：1886》** | Han 的技术——NDF 与 BRDF 卷积 → 存回 roughness 贴图 |
| **寒霜引擎** | 方差映射 + 多层法线贴图方差组合 |
| **虚幻引擎** | 类似方案 |
| **皮克斯动画电影** | LEAN 映射——协方差矩阵，比方差更精确但更贵 |

方差映射是**工业标准**：离线烘焙方差到 roughness mipmap → 运行时 GPU 自动选修正后的值 → 着色器零改动。
