## 第8章 光与颜色 —— 建立光的"语言"

第8章是全书的**度量衡基础**。在你能说"这个表面反射了多少红色光"之前，
你需要先有语言和单位。

### 8.1 三大度量学：从物理到人眼

| 学科 | 关注什么 | 核心单位 |
|------|---------|---------|
| **辐射度量学** | 纯粹物理量（不考虑人眼） | radiance $L$（$W/m^2 sr$） |
| **光度学** | 辐射量 × 人眼敏感度曲线 | luminance（$cd/m^2$，即 nit） |
| **色度学** | SPD（光谱功率分布）→ 三刺激值 | XYZ / xyY 坐标 |

**为什么 radiance 是渲染中最核心的量？** 因为相机/眼睛的传感器测量的就是
radiance——它是"沿着一条光线流动的能量密度"。而且 radiance 有一个美妙性质：
**不随距离衰减**（大气效应除外）。远处的墙和近处的墙，每个像素接受到的
radiance 是相同的，只是远墙占的像素更少。

### 8.1.3 色度学的关键洞察

SPD（光谱功率分布）是光的完整描述——一张以波长为 x 轴的曲线图。但人眼只有
三种视锥细胞，因此**不同的 SPD 可以产生完全相同的颜色感知**（同色异谱）。
这正是 RGB 渲染可行的理论基础：我们用三个数（RGB）就能近似表示光。

### 8.2 从场景 radiance 到屏幕像素的完整链条

```
场景radiance → 曝光 → 色调映射 → 显示编码 → 显示器 → 你的眼睛
(场景参考)     (仍是场景参考)   (显示参考)   (编码值)
```

这是很多人忽略的一步：**场景中计算的 radiance 和屏幕发出的 radiance 是两回事**。
色调映射（tone mapping）承担了二者之间的桥梁工作。

#### 色调映射的本质：图像再现，不是压缩

色调映射不是一个"把 HDR 塞进 LDR"的压缩问题，而是**图像再现**：
让显示器上的图像给人的**感知印象**尽可能接近原始场景。

这之所以可能，是因为人眼有**适应性**（compensates for absolute brightness），
同时也不完美：暗处感知对比度下降（Stevens 效应）、感知色彩度也下降
（Hunt 效应）。这些效应叠加显示器环境（房间照明、屏幕反射），意味着
显示参考图像必须提高对比度和饱和度，才能接近原始场景的感知。

这就是为什么 S 形（sigmoid/filmic）曲线普遍存在——它有三个区域：
- **暗部**：柔和抬升，保留阴影细节不被完全裁剪
- **中间调**：增强对比度，补偿 Stevens 效应
- **亮部**：柔和压缩，保留高光细节（而不是直接变成纯白）

常见实现：Reinhard → Duiker → Hable filmic → ACES（目前行业标准）→
寒霜引擎方案（在 ICtCp 空间中操作，保证不偏移色相）。

#### 曝光：在色调映射之前

计算曝光的最常见方法是通过**前一帧的亮度直方图**求中位数（比对数平均
更稳健，不受少量过亮像素影响）。更好的做法是用**入射光而不是像素亮度**
来确定曝光——这与摄影实践一致（18%灰卡原理），正在被越来越多的游戏采用。

#### 颜色分级：营造"氛围"

颜色分级在电影工业中是从《逃狱三王》（2000）开始的，现在游戏中也普遍使用。
核心工具是一个 3D LUT（颜色查找表）——LUT 的每个点是 RGB→RGB 的映射。
分为两种：**显示参考**（调映射后的像素，简单但质量较低）和**场景参考**
（调映射前的 radiance，质量更高，如《神秘海域4》采用的方式）。

#### HDR 显示

HDR 显示器常用的信号路径：HDR10（PQ 编码 + Rec.2020 色域，最普及）、
scRGB（Windows 专属，向后兼容 sRGB）、杜比视界。HDR 的色调映射往往
比 SDR 更保守，因为显示器本身能显示更大动态范围。

## 第9章 基于物理的着色 —— 表面如何反射光

第9章是全书**最核心的一章**。它回答了着色问题最根本的追问：光打到表面上，
到底发生了什么？

### 知识结构：从物理观测到数学工具

整个第9章的逻辑路径如下：

```

  ↓
9.3 BRDF 的数学定义（反射方程）
  ↓
9.5 菲涅尔效应（反射率随角度变化）
  ↓
9.6 微观几何（粗糙度的微观解释）
  ↓
9.7 微表面理论（把微观几何统计化：NDF + G₂ + F = BRDF）
  ↓
9.8 镜面 BRDF（微表面 BRDF 的精确形式：f_spec = F·G₂·D / (4|n·l||n·v|)）
  ↓
9.9 漫反射 BRDF（次表面散射的局部模型）
  ↓
9.10-12 特殊材质（布料、波动光学、分层材质）
  ↓
9.13 材质混合与过滤（不同类型材质的拼接 + 抗锯齿）
```

### 9.1 光的物理学：三幅"心理图像"

读这一节的关键是建立**对光的直觉**，记住三幅图：

- **干涉**（图9.3）：同相→相长干涉（能量集中），反相→相消干涉（能量抵消）。
  能量既不会凭空产生也不会消失，只在空间上重新分布。
- **均匀介质**（图9.5-9.8）：均匀排列的分子导致**相消干涉抑制了散射**，
  所以光在纯净介质中不会被散射——光沿直线传播。一旦出现不均匀（气泡、杂质、
  密度涨落），散射就发生了。
- **表面 = 折射率不连续面**（图9.9-9.11）：两个不同折射率的介质的交界面
  必然导致反射 + 折射。如果折射率相同？表面消失。（图9.11的珠子实验）

### 9.3 BRDF = 表面的"身份证"

$$L_o(\mathbf{p}, \mathbf{v}) = \int_{\Omega} f(\mathbf{l}, \mathbf{v}) L_i(\mathbf{l}) (\mathbf{n} \cdot \mathbf{l}) d\mathbf{l}$$

这个公式是第9-11章的**反复出现的主旋律**：

- $L_o$：我要计算的东西——从表面往某个方向发出了多少光
- $f(\mathbf{l}, \mathbf{v})$：BRDF——这是表面的"反射特性"，第9章的核心
- $L_i(\mathbf{l})$：入射光——第10章告诉我们它从哪来、长什么样
- $(\mathbf{n} \cdot \mathbf{l})$：余弦项——斜射的光"稀释"了，物理事实

**BRDF 有两个物理约束**：
1. **互易性**：交换入射和出射方向，结果不变（$f(\mathbf{l}, \mathbf{v}) = f(\mathbf{v}, \mathbf{l})$）
2. **能量守恒**：反射的能量 ≤ 入射的能量（$R(\mathbf{l}) \leq 1$）

最简单但依然很常用的 BRDF：**Lambertian** = $\rho_{ss}/\pi$。那个 $1/\pi$
是因为对半球范围的余弦项积分结果是 $\pi$——记住这个因子，它在后续章节中
会和光照公式中的 $\pi$ 因子相互抵消。

### 9.5 菲涅尔效应：边缘更亮

**$F_0$ 是材质的"镜面颜色"**——光线垂直入射时的反射率。

| 材料类型 | $F_0$ 范围 | 特点 |
|---------|-----------|------|
| 电介质（玻璃、皮肤、木材、塑料） | 0.02~0.06 | 无色，菲涅尔效应非常明显 |
| 金属 | 0.5~1.0 | 常有颜色（如金=黄色），吸收所有透射光 |
| 半导体 | 0.2~0.45 | 介于两者之间 |

**Schlick 近似**是实时渲染的主力：
$$F(\mathbf{n}, \mathbf{l}) \approx F_0 + (1-F_0)(1-\mathbf{n}\cdot\mathbf{l})^5$$

它是 $F_0$（正面颜色）和白色（掠射角）之间的插值，极其高效。

**金属度参数化**（metalness workflow）：用一个标量 $m$ 和一个 RGB 表面颜色
$\mathbf{c}_{surf}$ 同时编码 $F_0$ 和 $\rho_{ss}$——这是目前游戏工业的标准做法。

### 9.6-9.7 微观几何与微表面理论：把粗糙度变成数学

**核心思想**：我们不显式建模每个微小的表面凸起（那太贵了），而是统计它们的
法线分布。

**三大支柱**构成微表面 BRDF：

| 符号 | 含义 | 通俗理解 |
|------|------|---------|
| $D(\mathbf{m})$ | NDF 法线分布函数 | "有多少微表面法线恰好指向某个方向" |
| $G_2(\mathbf{l}, \mathbf{v})$ | 遮蔽-遮挡函数 | "从光线和人眼方向同时看到微表面的比例" |
| $F(\mathbf{h}, \mathbf{l})$ | 菲涅尔项 | "每个微表面反射了百分之多少的光" |

### 9.8.1 NDF：决定高光"长什么样"

| NDF | 特点 | 当前地位 |
|-----|------|---------|
| Beckmann | 最早，光学界标准 | 仍有使用，尾部较短 |
| Blinn-Phong | 计算最简单 | 移动端常用 |
| **GGX** (Trowbridge-Reitz) | **峰值尖锐 + 长拖尾** | **行业主流** |

GGX 的统治地位来自一个物理事实：**真实世界材质的高光普遍有很长的"拖尾"**，
即高光核心周围有微弱的模糊辉光。Beckmann 的尾部太短，GGX 最接近测量数据
（图9.37 对金属铬的拟合对比）。

**形状不变性（Shape Invariance）**——NDF 的一个关键特性：Beckmann 和 GGX
都具备，Blinn-Phong 不具备。Shape-invariant NDF 的粗糙度参数的效果等价于
"缩放微表面"（方程9.36），这导致其 Smith $\Lambda$ 函数只依赖一个变量
$a$（方程9.37）而不是两个——让 masking-shadowing 的计算和预过滤极大简化。
各向异性的推广（9.8.1 末尾）也依赖于形状不变性。

**各向异性 NDF**：用两个粗糙度参数 $\alpha_x$、$\alpha_y$ 控制不同方向的
模糊程度，用于拉丝金属、卷发等有方向性的材质。

### 9.8.2 多次反弹的表面反射

标准微表面 BRDF 只计算单次反弹——光打到微表面上反射一次后就消失了。
但现实中光线可能在微表面之间来回反射多次。这会损失能量，尤其对**粗糙金属**
影响大（金属的漫反射全靠多次反弹，本身没有次表面散射）。

**Imageworks 多次反弹修正项**（方程9.56）通过两个预计算来补偿：
- $R_{sF1}$：$F_0=1$ 时镜面 BRDF 的定向反照率（$32\times32$ 的 2D 查找表就够）
- $\bar{F}$：菲涅尔项的余弦加权平均（有解析公式）

### 9.9 漫反射：光滑 vs 粗糙的关键区分

很多人以为"粗糙表面就该用粗糙漫反射模型"。**错！** 正确判断依据是：
**次表面散射距离 vs 微观不规则性的相对大小**（图9.40）。

```
散射距离 < 不规则性  →  粗糙表面漫反射模型（逆反射效应显著）
散射距离 > 不规则性  →  光滑表面漫反射模型（无逆反射）
```

月球为什么有逆反射？因为从地球上观察，一块五英尺高的石头都算"微观几何"——
散射距离远小于这些不规则性。

**光滑表面模型**（9.9.3）：Lambertian → 乘 $(1-F)$ 的简单耦合（方程9.63）
→ Shirley 耦合模型（互易 + 能量守恒，方程9.64）→ **Kelemen-Szirmay-Kalos**
（方程9.65）——最完备方案，通过 $R_{spec}$ 与任意高光项耦合。

**粗糙表面模型**（9.9.4）：**Disney diffuse**（方程9.66）——最流行，
结合粗糙漫反射 $f_d$ 和次表面项 $f_{ss}$，共享镜面粗糙度参数。
Oren-Nayar（计算简单）、**Hammon 模型**（方程9.68，GGX + 两次反弹，
理论扎实、开销低）。

### 9.10-9.13：特殊材质与混合

- **布料**（9.10）：纤维织物不是普通的微表面——用逆高斯 NDF（天鹅绒）、
  微圆柱体模型（丝绸）、或经验边缘光项来模拟。
- **波动光学**（9.11）：1-100倍波长的不规则性→衍射和薄膜干涉（肥皂泡的
  彩虹色）。**薄膜干涉比你想象的更普遍**——即使是皮革表面的微变色也来自它。
- **分层材质**（9.12）：透明涂层（清漆）是最常见例子——光在涂层表面反射
  一次 + 在基层反射一次 = 双重高光。
- **过滤**（9.13）：mipmap 不能只平均法线！必须过滤整个 NDF 的分布。
  Toksvig / LEAN 映射 / 方差映射都是为了解决高光锯齿。

## 第10章 局部光照 —— 给表面的 BRDF 配上真实光源

第9章告诉你表面如何反射光。第10章回答：**入射光 $L_i(\mathbf{l})$
长什么样？**

### 从假光源到真光源

第9章用的是点光源/方向光（精确光源）——现实中不可能存在（立体角为0，
radiance 无穷大）。第10章开始来真的。

### 10.1 面光源：光源从"一个点"变成"一块面"

对面光源积分的基本形式：
$$L_o(\mathbf{v}) = \int_{\omega_l} f(\mathbf{l}, \mathbf{v}) L_l (\mathbf{n} \cdot \mathbf{l})^+ d\mathbf{l}$$

**向量 irradiance（Vector Irradiance）**：Lambertian 表面上任意形状的面光源
都可**精确**等效为点光源（方向 $\mathbf{l}_c$ = 光源的辐照度中心方向，
颜色 $\mathbf{c}_{light}$ = 按向量irradiance长度缩放）。唯一的误差来自
"地平线以下被遮挡的部分"——当光源部分位于表面背面时，点光源近似开始
不准确。对于光泽材质，无法这样精确转换。

**实际面光源的两种近似策略**：

| 策略 | 原理 | 代表 | 适用 |
|------|------|------|------|
| **粗糙度修正** | 光源扩散 → 等效粗糙度增加 | Karis, Mittring | 中等粗糙度 |
| **代表性点** | 光源上贡献最大的单一方向 | Picott, Karis, Drobot | 所有粗糙度 |

代表性点方法在实践中更受欢迎，因为它能保留锐利高光边缘（粗糙度修正方法
对光滑表面失效——没有办法让镜面"更光滑"来表示面光源的锐利反射）。

**LTC（线性变换余弦）**：用 $3\times3$ 矩阵拉伸余弦波瓣来拟合任意 BRDF
形状，然后**精确解析积分**（变换矩阵的逆作用于光源多边形定义域，把复杂
BRDF 的被积函数退化为简单的余弦波瓣）。比代表性点法更准确但略贵。

**光源形状**：球形（最简单，最常用）、管状/胶囊（荧光灯，Picott 给出漫反射
封闭解）、矩形/多边形（平面光源如柔光箱，LTC 最擅长处理）。

### 10.2-10.3 环境光照：光从四面八方来

月球没有大气层 → 没有天光 → 阴影是纯黑的。这就是环境光照的意义。

球面函数（$S^2$ 上的函数）的各种表示方法：

| 方法 | 用途 | 特点 |
|------|------|------|
| 表格形式（立方体贴图） | 存储环境 radiance | 最高频细节，GPU 硬件支持 |
| **球谐函数 (SH)** | 低频信号（irradiance） | 正交基、旋转不变、乘积=点积 |
| 球面高斯 (SG) | 中等频率（BRDF 近似） | 乘积和卷积有解析解 |
| **AHD** | 定向 irradiance | 极度紧凑（8个参数），游戏常用 |
| 环境立方体 | 快速替代方案 | 6个方向×cos²波瓣 |

### 10.4 环境映射：把球面函数变成贴图

| 映射方法 | 特点 |
|----------|------|
| 经纬度映射 | 最早，两极拉伸严重，有超越函数 |
| 球面映射 | 依赖观察方向（MatCap），简单 |
| **立方体贴图** | **当前最主流**——视角无关、采样均匀 |
| 八面体映射 | 正方形纹理，无缝隙问题 |

### 10.5 基于图像的高光照明（IBL）：环境贴图×光泽BRDF

核心问题：不能对整个环境贴图的每个像素都做一次 BRDF 积分——太贵了。

**分裂积分近似**（Split-Integral / Split-Sum）是目前工业标准：

$$\int f_{smf}(\mathbf{l}, \mathbf{v}) L_i(\mathbf{l}) (\mathbf{n} \cdot \mathbf{l}) d\mathbf{l} \approx \underbrace{\int D(\mathbf{r}) L_i(\mathbf{l}) (\mathbf{n} \cdot \mathbf{l}) d\mathbf{l}}_{\text{预过滤环境贴图（cube map mipmap）}} \cdot \underbrace{\int f_{smf}(\mathbf{l}, \mathbf{v}) (\mathbf{n} \cdot \mathbf{l}) d\mathbf{l}}_{\text{BRDF 积分 LUT（2D 纹理）}}$$

这个近似不是简单的"把积分拆成两项相乘"——两个被积函数**都包含**了镜面
波瓣 $D$，只是第一项假设 $\mathbf{n}=\mathbf{v}=\mathbf{r}$（径向对称），
第二项作为修正因子补回方向相关的部分。

第一项：对不同的粗糙度预过滤环境贴图，存在 cube map 的 mipmap 中。
粗糙度越高 → mipmap 层级越低（越模糊）。GPU 三线性插值可以混合相邻
粗糙度级别。

第二项：预计算为二维查找表——输入是 $(\cos\theta_v, \alpha_g)$。
Karis/Lazarov 的关键洞察：如果把 $F_0$ 从 Schlick 近似中提取出来，
$R_{spec}$ 可以分解为两项（$F_0$ 的线性组合），并且可以压缩为一个
双通道的 2D 纹理（RG 分别存储两个因子）。这一个 LUT 还能同时用在
**Kelemen-Szirmay-Kalos 耦合漫反射**（方程9.65）中——一举两得。

**一个实践经验**（Lagarde）：用反射方向访问预过滤环境贴图时，可以
按粗糙度把方向从反射向量稍微往法线方向倾斜——这补偿了地平线裁剪的
能量损失，减少了 IBL 和解析光源之间的高光不匹配。

## 第11章 全局光照 —— 物体之间相互照亮

**局部光照 vs 全局光照的分水岭**：局部光照的入射光 $L_i$ 是"从天上掉下来的"
（已知量）；全局光照的 $L_i$ 是"从其他表面反射过来的"（递归的未知量）。

### 11.1 渲染方程：光传输的终极公式

$$L_o(\mathbf{p}, \mathbf{v}) = L_e(\mathbf{p}, \mathbf{v}) + \int_{\Omega} f(\mathbf{l}, \mathbf{v}) L_o(r(\mathbf{p}, \mathbf{l}), -\mathbf{l}) (\mathbf{n} \cdot \mathbf{l})^+ d\mathbf{l}$$

与第9章的反射方程相比，只改了一项：$L_i(\mathbf{l})$ 变成了
$L_o(r(\mathbf{p}, \mathbf{l}), -\mathbf{l})$——即"从 $\mathbf{l}$ 方向
入射的光，等于另一表面向反方向发射的光"。

**这是递归的**。要算 A 点的颜色需要先知道 B 点的颜色，要算 B 点又需要先知道
C 点……无限递归。**路径追踪**（蒙特卡罗法）通过随机采样来近似这个积分；
**辐射度算法**把场景离散为面片，求解大型线性方程组。

### 11.2 Heckbert 的光路符号：统一视角

$L(D|S)^{*}E$ 概括了所有可能的光路。L=光源，E=眼睛，D=漫反射，S=镜面反射：
- $LE$：直接看到光源
- $LDE$：漫反射一次到眼睛（直接光照）
- $LSE$：镜面一次到眼睛（镜面反射）
- $LD^{*}E$：多次漫反射（辐射度算法模拟的）
- $LSSE$：镜面→镜面→眼睛（反射中的反射）
- $L(S|D)^{*}E$：**渲染方程**——光在到达眼睛之前可以反弹无限次

**实用价值**：给任何一种 GI 算法标注它能处理的路径，立刻可以看出它的能力
范围和局限性。

### 11.3 环境光遮蔽（AO）：全局光照的"零阶近似"

AO 是最简单的 GI 技术——物理近似很粗糙，但视觉回报巨大。

**出发点**：假设环境光是均匀的 $L_A$。那么 $E = \pi L_A$ ——每个点看起来
都一样亮。但现实中凹陷和缝隙会遮挡一部分入射方向。

**AO 系数**（方程11.8）：
$$k_A = \frac{1}{\pi} \int_{\Omega} v(\mathbf{l}) (\mathbf{n} \cdot \mathbf{l})^+ d\mathbf{l}$$

$k_A \in [0,1]$ = 未被遮挡半球的余弦加权比例。裂缝 → 低 $k_A$（暗）；
凸面 → $k_A=1$。同时计算**环境法线（bent normal）**：未遮挡方向的余弦加权
平均值——后续着色时替代几何法线，给出更正确的采样方向。

**Occlusion vs Obscurance**（11.3.2）：
- **Occlusion**：可见性函数 $v$ 是二值的（0或1）——无限远的光线是否被挡？
  封闭房间中所有射线都会撞到墙 → 全部黑掉
- **Obscurance**：用距离映射函数 $\rho(d)$ 替代——只关心有限距离内的遮挡。
  $d_{max}$ 需要手动调，物理上不正确，但在**感知上令人信服**。

**考虑相互反射**（11.3.3）：纯 AO 偏暗，因为没有计入二次反弹。简单修正
（方程11.13）：$k_A' = k_A / (1 - \rho_{ss}(1-k_A))$——假设遮挡方向的
radiance 和当前点的出射 radiance 相同，打破递归后用解析式近似。Jimenez
用离线路径追踪拟合了一个三次多项式来作更精确的修正。

**AO 的各种计算方法**：

| 方法 | 适用场景 | 代表 |
|------|---------|------|
| 预计算光线追踪 | 静态场景，最高质量 | 烘焙到纹理/顶点/体积 |
| 动态物体空间 | 刚体移动 | Bunnell 圆盘法、球体近似法 |
| **SSAO**（屏幕空间） | 任意动态物体 | Crytek → **HBAO** → **GTAO** |
| 符号距离场（SDF） | 解析/体素几何 | 锥形追踪（Wright, Crassin） |

GTAO（Jimenez，方程11.18）是目前质量最高的 SSAO——从深度缓冲构建高度场，
通过左右视界角解析积分，**不丢失余弦项**，与光线追踪 ground-truth 最接近。
每个像素只用 1 个样本 + 空间抖动 + 联合双边滤波 + 时域超采样，就能在 60FPS
下运行。

**AO 着色的限制**（11.3.7）：AO 只对**大面积光源和环境光照**有物理意义。
对精确光源（点光/方向光），可见性是非0即1的二值函数，用 AO 近似的效果
会很差——这种场景应该用阴影贴图。

### 11.4 定向遮蔽：AO 的升级版

AO 只知道"被遮挡了多少"，但不知道"被哪个方向遮挡了"——无法区分红色天光
被挡住和蓝色天光被挡住的情况。

**定向遮蔽** = 对可见性函数 $v(\mathbf{l})$ 的完整球面编码。常见编码方式：
视界映射、环境圆锥、球谐函数。

### 11.5 漫反射全局光照：光线在场景中弹跳

**表面预照明（Lightmapping）**：离线烘焙 irrdiance 到光照贴图。最早来自
Quake（1996年），至今仍大量使用。

**定向表面预照明**：存储带方向信息的 irrdiance（允许法线贴图）：
- 球谐函数（二阶或三阶）
- AHD（8 个参数：方向 + 环境光 + 高光颜色）——使命召唤系列
- H-basis（6 个系数可达三阶 SH 质量）

**PRT（预计算 radiance 传输）**：允许在运行时改变光照！
- 核心思想：预计算的是**传输函数**（场景对"单位光源"的响应），不是最终光照
- 运行时：实际光照 = 传输系数 × 当前光源强度，**线性叠加**即可
- Sloan 等人的原始 PRT（2002）：用球谐函数表示"单位光源"（每个 SH 系数
  是一个基光源），预计算场景对每个基光源的响应（传输向量/矩阵）。
  三阶 SH → $9\times9$ 矩阵 → 81 个存储值/**点** → 存储爆炸。
- 解决方案：PCA 压缩 + 聚类 → 把 625 维压缩到 256 维（还是太高，但思路开启了）。
- **实战变体**：用少数离散方向（如环境立方体的 6 个方向）作为基光源 → 开销大幅降低
- 用于《孤岛惊魂3/4》《刺客信条4》《命运》等大量游戏

**Enlighten 系统**（11.5.3 末尾）：另一种思路——不是预计算光源→场景的传输，
而是预计算**场景表面之间的传输**（类似辐射度的形状因子）。运行时把直接光照
"注入"到场景表面，然后预计算的传播矩阵把能量扩散出去。这种方式允许任意位置
的光源。Unity 早期版本内置了 Enlighten。

**光照贴图的存储问题**（11.5.4）：
- 每个三角形需要在光照贴图上有唯一位置 → 场景被切割成 chart，再打包到一个纹理
- **接缝**（seam）是经典难题：chart 边缘两侧的光照不一致产生视觉不连续。
  解决：把接缝放在不可见面（手动），或者后处理沿边缘修正纹素值
- **颜色溢出**（chart 之间的过滤占用空间重叠）→ 需要正确计算每个 chart
  的"排水沟"，或使用 grid-preserving parameterization

### 11.6 镜面 GI：反射和光泽间接光

镜面 GI 通常不靠离线烘焙（因为依赖于观察方向），而是通过：
- 局部环境贴图（probe）+ 动态更新
- 平面反射（地板/水面的反射）
- 屏幕空间反射（SSR）

### 11.7 统一方法的未来

路径追踪 + 降噪是趋势。随着 RTX 硬件的普及，离线质量的 GI 正逐步进入
实时渲染领域。

## 四章的核心关联（必读）

这四章的关联不是"教科书式"的先后顺序，而是**逻辑推断链**。每一个问题的
答案自然引出下一个问题。

### 关联链：一个问题的答案引出下一个问题

```
问题① 光是什么？怎么量？
  Ch8 回答：光 = radiance（辐射度量学）+ 三种视锥细胞（色度学）→ 用 RGB 近似

问题② 表面收到光后怎么反射？
  Ch9 回答：BRDF —— 一个 4D 函数，描述每个入射→出射方向的反射比
          核心材质属性：F₀（镜面颜色）+ ρ_ss（次表面反照率）+ α（粗糙度）

问题③ 实际光源不是理想点光，怎么处理？
  Ch10 回答：面光源积分（代表性点/LTC）+ 环境贴图（IBL/Split-Sum）
          L_i(l) 不再是单一方向，而是整个半球上的分布

问题④ 入射光不只来自光源，还来自其他表面——怎么办？
  Ch11 回答：渲染方程 = 反射方程 + 递归项
          AO：对遮挡的最简近似
          定向遮蔽：对可见性 v(l) 的编码
          PRT / Lightmap：预计算光线传输
          路径追踪：ground-truth（太慢，但概念上完美）
```

### 具体到公式的连接点

| 公式 | 出现处 | 意义 |
|------|-------|------|
| $L_o(\mathbf{v}) = \int f(\mathbf{l},\mathbf{v}) L_i(\mathbf{l}) (\mathbf{n}\cdot\mathbf{l}) d\mathbf{l}$ | Ch9 方程9.4 | **反射方程**——一切着色计算的起点 |
| $f_{spec} = F G_2 D / (4\|\mathbf{n}\cdot\mathbf{l}\|\|\mathbf{n}\cdot\mathbf{v}\|)$ | Ch9 方程9.34 | **微表面镜面 BRDF**——三个因子撑起所有高光计算 |
| $F \approx F_0 + (1-F_0)(1-\mathbf{n}\cdot\mathbf{l})^5$ | Ch9 方程9.16 | **Schlick 菲涅尔**——$F_0$ 贯穿所有章节 |
| $L_{split} = \int D(\mathbf{r})L_i(\mathbf{n}\cdot\mathbf{l})d\mathbf{l} \cdot \int f_{smf}(\mathbf{n}\cdot\mathbf{l})d\mathbf{l}$ | Ch10 方程10.39 | **Split-Sum**——Ch9 的微表面理论 × Ch10 的环境贴图 |
| $L_o(\mathbf{p},\mathbf{v}) = L_e + \int f L_o(r(\mathbf{p},\mathbf{l}),-\mathbf{l}) (\mathbf{n}\cdot\mathbf{l})^+ d\mathbf{l}$ | Ch11 方程11.2 | **渲染方程**——Ch9 反射方程 × 光线投射递归 |
| $k_A = \frac{1}{\pi} \int v(\mathbf{l})(\mathbf{n}\cdot\mathbf{l})^+ d\mathbf{l}$ | Ch11 方程11.8 | **AO 系数**——渲染方程的可见性简化 |

### 微表面理论在 Ch9-Ch11 中的传递

```
Ch9  构建：每个微表面是完美镜面 → f_spec = F·G₂·D / denominator
      D = GGX/Beckmann （法线分布的形状选择性）

Ch10 应用：微表面 BRDF × 环境贴图积分
      Split-Sum 把 D 径向对称化（n=v=r）→ cube map mipmap 预过滤
      BRDF LUT 补偿方向相关的部分

Ch11 扩展：镜面 GI = 动态更新环境贴图 + 与微表面 BRDF 卷积
      定向遮蔽中对 v(l) 的编码也需要和微表面 BRDF 的波瓣做积分
```

### 工程降级链：每步都知道"牺牲了什么"

```
路径追踪（蒙特卡洛）            Ch11 §11.2.2
  │ 完全求解渲染方程，所有光路类型     牺牲：性能（数小时/帧）
  ▼
PRT / 辐射度 / Lightmap          Ch11 §11.5
  │ 预计算传输，运行时线性组合        牺牲：仅支持漫反射传输（L D* SE）
  ▼
IBL（Split-Sum）                  Ch10 §10.5
  │ 环境贴图 + BRDF 近似积分        牺牲：光源无限远假设，无局部遮挡
  ▼
面光源 + LTC/RP                    Ch10 §10.1
  │ 对单个面光源精确积分            牺牲：仅处理直接光，无环境/间光
  ▼
精确光源 + AO                      Ch9 + Ch11 §11.3
  最简单的实时方案                   牺牲：无面光源效果，AO 对点光不准确
```

每一步降级都保留了上一步的核心思想——例如 Split-Sum 虽然拆开了积分，但
两个积分项**都**包含 $D$（不是粗暴地拆成两半），保留了微表面理论的核心结构。

### 金属 vs 电介质二分类贯穿全部决策

```
金属：F₀ 高（≥0.5）        → 镜面项主导，无漫反射
      ρ_ss = 0             → 颜色 = F₀ 的光谱
      多次反弹补偿 f_ms 重要 → Ch9 §9.8.2
      IBL 高光效果明显       → Ch10 §10.5

电介质：F₀ 低（≤0.06）     → 镜面项弱，菲涅尔效应明显
        ρ_ss 主导外观       → 颜色 = ρ_ss 的光谱
        漫反射 + AO 效果重要 → Ch11 §11.3
        需要耦合漫反射       → Ch9 §9.9（Kelem-Sz-Kalos 或 Disney）
```

这个分类决定着色器中的分支逻辑、纹理存储格式（金属度工作流的 $m$ 标量
+ $\mathbf{c}_{surf}$ 值）、以及 GI 计算中对哪种项的预算分配。

### 关键数值记忆点

- **电介质 $F_0$ ≈ 0.04**（最常用的默认值，比大部分电介质略高但不离谱）
- **金属 $F_0$ ≥ 0.5**（铁约 0.56（线性），银最高约 0.95）
- **不存在 0.02 < $F_0$ < 0.2 的"中间"材质**（半导体范围内的晶体硅除外）
- **Lambertian BRDF = $\rho_{ss}/\pi$**——记住 $1/\pi$，因为光照公式中的
  $\pi$ 因子会与它抵消
- **AO 不适用于精确光源**——可见性函数对点光是二值的，不能用平滑的 AO 近似
- **GGX 粗糙度映射**：$\alpha_g = r^2$（$r$ 是用户 0-1 的滑条值）—
  视觉更线性
